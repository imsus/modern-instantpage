Why instant.page?

⚡

Speculation Rules API

Uses the modern Speculation Rules API when available, with automatic fallback to link prefetch for older browsers.

🎯

Smart Prefetching

Only preloads when there's a good chance of navigation — on hover, mousedown, touch, or viewport visibility.

📱

Mobile Optimized

Touch-aware prefetching, respects data saver mode, and adapts to network conditions automatically.

🔒

Privacy First

Only preloads HTML documents. No tracking, no cookies, no external requests. Respects user privacy.

🚀

Priority Hints

Uses fetchPriority="high" for user-initiated prefetches to ensure fast loading without blocking other resources.

📦

Tiny Footprint

Under 2KB gzipped. Zero dependencies. Loads after everything else via scheduler.postTask().

How it works

1

User hovers over a link

On desktop, when a user hovers for 65ms, there's a 50% chance they'll click. That's when we start preloading.

2

Page preloads in background

The browser fetches the next page using Speculation Rules or link prefetch, leaving 300ms+ for completion.

3

User clicks — page is ready

The page loads instantly from cache. The user perceives it as immediate (<100ms).

Configuration

Configure via data attributes on <body>

Attribute Description
data-instant-intensity="mousedown" Preload on mousedown instead of hover (80ms average)
data-instant-intensity="mousedown-only" Mousedown preload on desktop only, no mobile preloading
data-instant-intensity="viewport" Preload links as soon as they're visible (mobile only)
data-instant-intensity="viewport-all" Preload all visible links on any device
data-instant-intensity="150" Custom hover delay in milliseconds
data-instant-allow-query-string Allow preloading pages with query strings
data-instant-allow-external-links Allow preloading external links
data-instant-whitelist Only preload links with data-instant attribute
data-instant-mousedown-shortcut Trigger click on mousedown for instant navigation
data-no-instant Blacklist specific links from preloading
